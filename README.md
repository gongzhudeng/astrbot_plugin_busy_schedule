# 灵犀 · AI忙碌时段管理插件

> 作者：灵犀 | 仓库：[astrbot_plugin_busy_schedule](https://github.com/gongzhudeng/astrbot_plugin_busy_schedule) | 主页：[gongzhudeng](https://github.com/gongzhudeng)

让AI拥有真实的生活节奏，不再是一个24小时在线的机器人。

## 功能特性

### 核心功能
- **智能忙碌时段计算**：根据日程安排自动计算一天的忙碌时段
- **消息拦截合并**：忙碌时段内的消息会被拦截并合并，忙完后统一处理
- **提前通知**：进入忙碌前会提前告知用户，避免突然消失
- **特殊关键词唤醒**：支持设置关键词让AI立即响应
- **聊天保护**：用户正在聊天时不会进入忙碌状态
- **动态日程调整**：可选功能，根据聊天上下文动态调整忙碌时段

### 日程生成
- 每日自动生成穿搭和日程安排
- 每个活动标记忙碌状态（可回消息/不可回消息）
- 支持创意池随机选择，增加多样性
- 历史日程参考，避免重复

### 提示词注入
- 正常时段：注入当前穿着、活动状态
- 忙碌时段：标记当前忙碌状态
- 忙碌结束：注入提示说明忙碌时段收到的消息

## 使用方法

### 命令列表

| 命令 | 别名 | 说明 |
|------|------|------|
| `查看日程` | `busy show` | 查看今日日程和忙碌时段 |
| `重写日程` | `busy renew` | 重写今日日程（可附加补充要求） |
| `忙碌状态` | `busy status` | 查看当前忙碌状态 |
| `忙碌帮助` | `busy help` | 显示帮助信息 |

### 配置项

#### 基础设置
- **启用插件**：总开关
- **日程生成时间**：每日生成日程的时间（默认 07:00）
- **LLM供应商**：指定生成日程和计算忙碌时段的LLM

#### 忙碌时段设置
- **提前通知时间**：进入忙碌前多少分钟通知用户
- **聊天保护时间**：用户多少分钟没发消息才能进入忙碌
- **唤醒后冷却时间**：被关键词唤醒后多少分钟重新计算忙碌

#### 睡眠设置
- **基础睡眠时间**：默认睡眠时段（如 23:00-07:00）
- **动态睡眠调整**：根据用户聊天情况动态调整睡眠时间
- **无消息睡眠阈值**：用户多少分钟没消息可以进入睡眠

#### 关键词设置
- **唤醒关键词**：触发立即响应的关键词列表
- **匹配模式**：包含模式 / 完全匹配模式

#### 消息合并设置
- **最大消息数**：达到此数量立即发送合并消息
- **延迟发送时间**：最后一条消息后多少秒发送

#### 动态调整设置
- **启用动态调整**：是否根据聊天上下文动态调整忙碌时段
- **调整冷却时间**：两次调整之间的最小间隔

## 提示词注入说明

### 正常时段注入
```
<character_state>
## 当前状态
- 穿着：[今日穿搭描述]
- 当前活动：[当前日程活动]
- 时间：[当前时间]
</character_state>
```

### 忙碌时段注入
```
<character_state>
## 当前状态
- 穿着：[今日穿搭描述]
- 当前活动：[当前日程活动]
- 时间：[当前时间]
- **当前处于忙碌状态，正在[忙碌原因]**
</character_state>
```

### 忙碌结束注入
```
[忙碌时段提示]
以下消息是你在忙碌时段（[开始时间]-[结束时间]）收到的用户消息：
[合并的消息内容]
请回复用户，并说明你刚才在做什么。
[/忙碌时段提示]
```

## 数据存储

- 日程数据：`schedule_data.json`
- 忙碌时段数据：`busy_periods.json`
- 消息队列：内存中存储，插件重启后清空

## 注意事项

1. 插件重启后，当天的日程数据会保留，但消息队列会清空
2. 动态调整功能会消耗额外的token，建议根据需要开启
3. 睡眠保护功能确保用户聊天时不会突然进入睡眠状态
